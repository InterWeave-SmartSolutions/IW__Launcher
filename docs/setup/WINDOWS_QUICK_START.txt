                    IW_IDE WINDOWS QUICK START
                      (No WSL Required)
================================================================================

PREREQUISITES
-------------
NONE! This package includes everything needed:
  - Bundled Java Runtime (JRE 8) in the 'jre' folder
  - Apache Tomcat 9.0.83 pre-configured
  - All required libraries and dependencies

No system Java installation required.

================================================================================
                        QUICK START
================================================================================

Option A: Start Everything (Recommended)
----------------------------------------
1. Double-click: START.bat
2. Wait 15-30 seconds for Web Portal to initialize
3. IDE will launch automatically
4. Open browser to: http://localhost:9090/iw-business-daemon/IWLogin.jsp

Option B: Start Components Individually
---------------------------------------
Start IDE only:        Double-click scripts\start_ide.bat (or iw_ide.exe)
Start Web Portal only: Double-click scripts\start_webportal.bat
Stop Web Portal:       Double-click scripts\stop_webportal.bat

================================================================================
                        TROUBLESHOOTING
================================================================================

Issue: "Port 9090 already in use"
  -> Another app is using port 9090
  -> Either stop that app, or edit web_portal\tomcat\conf\server.xml to change port

Issue: Tomcat window closes immediately
  -> Run from Command Prompt to see error messages
  -> Check web_portal\tomcat\logs\catalina.out for details

Issue: "localhost refused to connect" in browser
  -> Wait longer for Tomcat to start (can take 30+ seconds)
  -> Check if java.exe is running in Task Manager
  -> Check Windows Firewall isn't blocking port 9090

Issue: IDE won't start
  -> Ensure 'jre' folder exists in IW_Launcher directory
  -> Check that jre\bin\javaw.exe exists

Issue: "Java not found" errors
  -> The bundled JRE should be used automatically
  -> If problems persist, verify jre folder wasn't deleted
