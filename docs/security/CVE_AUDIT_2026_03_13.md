## 2. Top 10 Most Critical Security Concerns

### RANK 1 — CRITICAL: Apache Xerces-J 2.7.1 (2005) + Xerces 1.x (2001)
**Location**: Both `iwtransformationserver` and `iw-business-daemon`
**Risk**: XML External Entity (XXE) injection, billion laughs DoS, XML bomb
**Known CVEs**:
- **CVE-2012-0881** (CVSS 7.5 HIGH) — Xerces-J hash collision DoS
- **CVE-2013-4002** (CVSS 7.1 HIGH) — XML parsing DoS via external parameter entities before 2.12.0
- **CVE-2022-23437** (CVSS 6.5 MEDIUM) — Infinite loop parsing crafted XML (prior to 2.12.2)
**Remediation**: IMMEDIATE — upgrade to Xerces 2.12.2+

### RANK 2 — CRITICAL: Apache Xalan-J 2.7.0 / XSLTC 2.7.0 (2005)
**Location**: `iwtransformationserver` (core transformation pipeline) and `iw-business-daemon`
**Risk**: XSLT injection, arbitrary code execution via crafted stylesheets
**Known CVEs**:
- **CVE-2014-0107** (CVSS 7.5 HIGH) — Security restriction bypass in TransformerFactory
- **CVE-2022-34169** (CVSS 7.5 HIGH) — Integer truncation allows arbitrary bytecode execution
**Remediation**: IMMEDIATE — upgrade to Xalan 2.7.3+ or use JDK 8u341+ patched version

### RANK 3 — CRITICAL: MySQL Connector/J 5.1.15 (2011)
**Location**: `iwtransformationserver`
**Risk**: Remote code execution via deserialization, authentication bypass
**Known CVEs**:
- **CVE-2017-3523** (CVSS 8.5 HIGH) — RCE via `autoDeserialize` connection property
- **CVE-2018-3258** (CVSS 8.8 HIGH) — Deserialization RCE
- **CVE-2019-2692** (CVSS 7.5 HIGH) — Privilege escalation
**Remediation**: HIGH — replace with 8.0.33 (already in `tomcat/lib/`) or 8.4.x

### RANK 4 — CRITICAL: Apache Axis 1.2.1 (2005)
**Location**: `iwtransformationserver`
**Risk**: SSRF, RCE via deserialization, XXE
**Known CVEs**:
- **CVE-2023-51441** (CVSS 7.2 HIGH) — SSRF via crafted XML
- **CVE-2019-0227** (CVSS 7.5 HIGH) — SSRF via ServiceFactory
- **CVE-2014-3596** (CVSS 5.8 MEDIUM) — SSL certificate validation bypass
**Remediation**: HIGH — Axis 1.x is abandoned. Mitigate with network egress restrictions.

### RANK 5 — HIGH: Oracle OJDBC 10.2.0.1.0 (2006)

## 4. Risk Assessment

**Overall Dependency Posture: HIGH RISK**

The `iwtransformationserver` is a legacy engine from ~2005 with 130 vendor JARs, most from 2001-2005. The majority have known CVEs. However:

**Mitigating factors:**
1. The transformation server is behind Tomcat 9.0.83 and does not directly face the internet
2. XSLT files are admin-controlled (not user-uploaded)
3. External XML input comes from known integration partners (Salesforce, Creatio, QuickBooks)
4. The `iw-business-daemon` webapp has modern security filters (CSRF, rate limiting, CSP, auth)
5. Supabase has RLS on all 14 tables

**Recommendation:** Prioritize Xerces/Xalan upgrade (Ranks 1-2) as these process XML input. Replace MySQL 5.1.15 with the existing 8.0.33 (trivial). Remove test JARs. Accept remaining risks as legacy technical debt with network-level mitigation.
